1.3uploads：导入上传的csv数据存放在此

1.5vene：虚拟环境文件（可以删了）
1.6app.py：启动文件
1.7config.py：后端配置文件(mysql,jwt配置) jwt:身份验证工具
1.8heart_disease.db: 貌似是数据转换之后的心脏病数据
1.9requirement.txt：库文件
以上最重要的是models和routes文件
routes存储的就是函数编写的功能（后面细讲）

2.frontend：前端文件

2.2public：模板
2.3src：vue框架文件都在里面 这文件就是vue框架，其中最重要的就是views文件，里面存放的就是前端的编写，其他文件夹都是存放的一些引用（不重要）

其他文件不重要，但是不能删，都是一些配置信息的填写

3.heat_disease_predicetion.egg-info :没用可以删了

6.venv可以删了

7.下面的其他文件不重要


下面我着重说一下函数功能的实现

1.最重要的心脏病预测的功能 

1. 用户访问预测页面 Prediction.vue，提供用户输入界面，用于填写患者信息和健康指标。
2. 页面加载时，前端通过 axios向后端发送请求（http://localhost:5003/api/patients），获取现有患者列表。
3. 后端返回患者列表数据，前端将其展示在下拉菜单中，供用户选择现有患者或手动输入新患者信息。
4. 用户填写患者信息（如姓名、性别、年龄等）或选择现有患者。
5. 用户填写健康状况和健康指标（如总胆固醇、血压、血糖等）。
6. 用户点击进行预测按钮，触发 submitForm`方法。
7. submitForm方法验证表单数据是否完整。
   如果表单数据不完整，提示用户“请填写所有必填字段”。
   如果表单数据完整，继续下一步。
8. 前端通过 axios向后端 API 发送预测请求，调用 predictionService.predict方法。请求携带用户填写的表单数据（患者信息和健康指标）。
9. 后端 Flask 应用接收预测请求。
10. 调用 HeartDiseasePredictionAPI类中的 predict方法，根据输入的健康指标进行预测。
11. predict 方法返回预测结果，包括风险概率和风险等级。
12. 前端接收后端返回的预测结果。
13. 将预测结果显示在页面上，包括风险概率和风险等级。
14. 前端调用 savePredictionToDatabase方法，将预测结果保存到数据库。
15. savePredictionToDatabase 方法通过axios向后端发送请求（http://localhost:5003/api/predictions），将预测结果存储到数据库中。


2.查看预测记录
1. 用户登录后进入预测记录页面 PredictionHistory.vue， 页面加载时，前端通过axios向后端发送请求，获取历史预测记录。请求携带分页参数。
2. 后端接收请求，调用 get_predictions方法，从数据库中查询预测记录。查询结果按预测时间降序排列，并支持分页。后端返回预测记录列表，包括每条记录的 ID、患者姓名、预测时间、风险等级等信息。
3.. 前端接收响应数据，将预测记录列表展示在表格中。
#查看详情功能
1.用户在预测记录表格中选择某条记录，点击“查看详情”按钮。前端通过 axios 向后端发送请求（http://localhost:5003/api/predictions/<prediction_id>），获取该记录的详细信息。请求携带预测记录的 ID。
2.后端接收请求，调用 get_prediction方法，从数据库中查询指定 ID 的预测记录。后端返回预测记录的详细信息，包括患者信息、健康指标、预测结果等。
3. 前端接收响应数据，跳转到详细页面，展示该记录的详细信息。
#删除
1.用户在预测记录表格中选择某条记录，点击“删除”按钮。前端弹出确认对话框，提示用户是否确定删除该记录。如果用户确认删除，前端通过 axios`向后端发送删除请求。请求携带预测记录的 ID。
2. 后端接收请求，调用delete_prediction方法，从数据库中删除指定 ID 的预测记录。 后端返回删除操作的结果。
3. 前端接收响应数据，提示用户删除成功，并刷新预测记录列表。
#刷新
1. 用户点击“刷新数据”按钮，触发 refreshData方法。
2. 前端重新调用fetchData方法，从后端获取最新的预测记录列表。
3. 前端更新表格中的数据，展示最新的预测记录。


2.管理功能

2.1用户登录

1.用户访问登录页面 login.vue，用户填写用户名、密码、验证码，并选择用户角色（医生或普通用户）。用户点击“登录”按钮，触发 handleLogin 方法。handleLogin 方法验证验证码是否正确。
	如果验证码错误，提示用户“验证码输入错误”，并刷新验证码。
	如果验证码正确，使用 axios 向后端发送登录请求，并携带上用户名、密码和用户角色。
2.后端 根据User 和patinent中的 check_password 方法验证密码是否正确。如果用户名或密码错误，返回错误信息。如果登录成功，后端返回用户信息和访问令牌（access_token）。
3.前端使用 Vuex 的 login 方法保存用户信息和令牌。根据用户角色（医生或普通用户），跳转到对应的页面：
	医生跳转到 /doctor，普通用户跳转到 /user。
	如果登录失败，提示用户“登录失败，请检查用户名或密码”，并刷新验证码。

2.2用户注册
1. 用户访问注册页面 register.vue。
2. 用户填写用户名、密码、确认密码、验证码，并选择用户角色（医生或普通用户）。
3. 用户点击“注册”按钮，触发 handleRegister方法。
4. `handleRegister` 方法验证两次输入的密码是否一致。
   - 如果不一致，提示用户“两次输入的密码不一致”，并刷新验证码。
5. 如果密码一致，验证验证码是否正确。
   - 如果验证码错误，提示用户“验证码输入错误”，并刷新验证码。
6. 如果验证码正确，使用 axios向后端发送注册请求,请求携带用户名、密码和用户角色。
7. 后端 User 模型通过 set_password`方法设置用户密码，并保存用户信息。
8. 如果注册成功，提示用户“注册成功”，并跳转到登录页面 /login。
9. 如果注册失败，提示用户“注册失败，请重试”，并刷新验证码。

3.心脏病信息展示功能
1. 用户访问 dashboard.vue页面。
2. 页面加载时，显示心脏病的危害、预防措施和健康食物推荐。
3. 用户点击某个健康食物卡片，触发 showFoodDetail方法。
4. showFoodDetail方法显示该食物的详细信息（如健康益处、推荐食用量和营养成分）。
5. 用户点击关闭按钮，关闭食物详情弹窗。


4.特征重要性分析功能
1. 用户访问 featureimportance.vue页面。
2. 页面加载时，调用 fetchData方法从后端获取特征重要性数据（http://localhost:5003/api/feature_importance）。
3. 如果获取数据成功，使用 echarts绘制特征重要性柱状图。
4. 将特征重要性数据展示在表格中，包括特征名称、描述和重要性得分。
5. 如果获取数据失败，显示错误信息。
6. 用户点击“刷新数据”按钮，重新调用 fetchData方法更新数据。
